### GitHub Actions

- **Workflow Authoring:** Expert knowledge of YAML syntax, workflow triggers (push, pull_request, schedule, workflow_dispatch, repository_dispatch), conditional execution, and matrix strategies with include/exclude patterns
- **Expressions & Contexts:** Mastery of workflow expressions, context objects (`github`, `env`, `job`, `steps`, `runner`, `secrets`, `inputs`), functions (`contains`, `toJSON`, `fromJSON`), and conditional logic using `if:` expressions
- **Job Orchestration:** Mastery of job dependencies, concurrency controls, parallel execution, artifact passing, and workflow composition via reusable workflows and workflow_call
- **Runner Management:** Deep understanding of GitHub-hosted vs self-hosted runners, runner groups, labels, scaling strategies, ephemeral runners, and larger runners configuration
- **Secrets & Variables:** Secure handling of repository/organization/environment secrets, OIDC authentication, environment files (`GITHUB_ENV`, `GITHUB_OUTPUT`, `GITHUB_PATH`), and context-aware variable management
- **Action Development:** Proficiency in creating custom composite, JavaScript, and Docker container actions with proper input/output handling, versioning, and understanding trade-offs between action types
- **Caching & Optimization:** Expert implementation of dependency caching, build artifact caching, matrix strategy optimization, and strategies to minimize workflow execution time and billable minutes
- **Security Hardening:** Implementation of least-privilege permissions, pinned action versions (SHA), secrets scanning, dependency review, fork PR restrictions, and supply chain security (Sigstore, SLSA)
- **Debugging & Troubleshooting:** Proficiency with workflow debugging using `ACTIONS_STEP_DEBUG`, `act` for local testing, workflow visualization, run logs analysis, and step annotation commands
- **Marketplace Integration:** Effective use of community actions, evaluation criteria for third-party actions, and contribution to the GitHub Actions ecosystem
