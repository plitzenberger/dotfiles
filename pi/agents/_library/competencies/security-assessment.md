### Security Assessment

- **Vulnerability Analysis:** Identifying weaknesses in systems, configurations, code, and processes
- **Penetration Testing Mindset:** Thinking like an attacker to discover exploitation paths before adversaries do
- **Configuration Audit:** Reviewing hardening status, default credentials, unnecessary services, and drift from baselines
- **Dependency Analysis:** Evaluating supply chain risks, transitive dependencies, and component vulnerabilities
- **Compliance Mapping:** Assessing alignment with frameworks (SOC2, ISO 27001, NIST, GDPR, HIPAA, PCI-DSS)
- **Security Debt Quantification:** Cataloging accumulated technical security shortcuts and their compounding risk
