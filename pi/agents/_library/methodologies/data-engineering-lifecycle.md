### Data Engineering Lifecycle

A systematic approach to data engineering projects:

#### Phase 1: Discovery & Requirements

- Identify data sources and consumers
- Define data quality requirements and SLAs
- Establish success metrics and acceptance criteria
- Document compliance and governance constraints
- Assess infrastructure and tooling landscape

#### Phase 2: Architecture & Design

- Design data models and schema evolution strategy
- Define pipeline architecture and processing patterns
- Plan for scalability, fault tolerance, and observability
- Establish data contracts between producers and consumers
- Create technical specifications and ADRs (Architecture Decision Records)

#### Phase 3: Implementation & Testing

- Implement data pipelines with idempotency and atomicity
- Build data quality validation layers
- Create comprehensive test suites (unit, integration, data quality)
- Implement monitoring, alerting, and logging
- Document code, configurations, and runbooks

#### Phase 4: Deployment & Operations

- Deploy with infrastructure as code (IaC)
- Implement CI/CD for data pipelines
- Configure monitoring dashboards and alerts
- Establish incident response procedures
- Train operations team on runbooks

#### Phase 5: Optimization & Evolution

- Monitor performance metrics and costs
- Identify and resolve bottlenecks
- Implement schema evolution as requirements change
- Continuously improve data quality metrics
- Archive or sunset deprecated pipelines
