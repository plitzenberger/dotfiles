### Backend Development Lifecycle

A systematic approach to backend engineering projects:

#### Phase 1: Requirements & Discovery

- Gather functional and non-functional requirements
- Define API contracts and data models
- Identify integration points and dependencies
- Establish performance targets (latency, throughput, availability)
- Document security and compliance requirements
- Assess infrastructure constraints and preferences

#### Phase 2: Architecture & Design

- Design system architecture and component interactions
- Define database schema and data access patterns
- Plan API endpoints and authentication mechanisms
- Design for scalability, reliability, and observability
- Create technical specifications and ADRs
- Review design with stakeholders and address concerns

#### Phase 3: Implementation

- Set up development environment and CI/CD pipeline
- Implement core business logic with appropriate abstractions
- Build API endpoints with validation and error handling
- Implement database access layer and migrations
- Add authentication, authorization, and security controls
- Write unit and integration tests alongside code

#### Phase 4: Testing & Quality

- Execute comprehensive test suites (unit, integration, e2e)
- Perform load testing and capacity planning
- Conduct security review and vulnerability assessment
- Validate API contracts and backward compatibility
- Review code quality and address technical debt
- Document known limitations and edge cases

#### Phase 5: Deployment & Operations

- Deploy with infrastructure as code (IaC)
- Configure monitoring, alerting, and logging
- Set up on-call procedures and runbooks
- Perform gradual rollout with feature flags
- Monitor system health and performance metrics
- Gather feedback and plan iterations
