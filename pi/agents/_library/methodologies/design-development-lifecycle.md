### Design Development Lifecycle

#### Phase 1: Discovery & Foundation

1. **Design Guidelines Review**
   - Analyze provided design guidelines (principles, colors, typography, iconography)
   - Map design tokens to implementation requirements
   - Identify gaps or ambiguities requiring clarification
   - Document platform-specific adaptations needed

2. **Requirements Understanding**
   - Review user stories and acceptance criteria
   - Understand user context and journey placement
   - Identify accessibility requirements and constraints
   - Clarify device and platform targets

#### Phase 2: Component Design

1. **Component Architecture**
   - Define component hierarchy and composition patterns
   - Map to design system tokens and variants
   - Document props, states, and interaction patterns
   - Create component API specifications

2. **Visual Design**
   - Apply design guidelines to concrete UI elements
   - Design responsive breakpoints and adaptive layouts
   - Specify spacing, sizing, and visual hierarchy
   - Document light/dark mode variations

#### Phase 3: Interaction & Accessibility

1. **Interaction Design**
   - Define micro-interactions and transitions
   - Specify loading, empty, and error states
   - Document gesture behaviors and haptic feedback
   - Create animation specifications

2. **Accessibility Audit**
   - Verify color contrast compliance
   - Define screen reader labels and reading order
   - Specify keyboard navigation patterns
   - Document reduced motion alternatives

#### Phase 4: Implementation Support

1. **Developer Handoff**
   - Provide detailed specifications with measurements
   - Document edge cases and boundary conditions
   - Create implementation notes for complex interactions
   - Supply asset exports and token mappings

2. **Quality Assurance**
   - Review implemented components against specifications
   - Verify accessibility compliance on real devices
   - Test across target platforms and screen sizes
   - Identify and document refinements needed
