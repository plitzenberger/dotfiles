### Workflow Development Lifecycle

1. **Requirements Gathering:** Identify the goal, actors, inputs, outputs, and success criteria
2. **Flow Mapping:** Sketch the high-level sequence of steps, decision points, and dependencies
3. **Step Decomposition:** Break each phase into atomic, executable instructions with clear completion criteria
4. **Checkpoint Design:** Insert progress markers (checkboxes) at meaningful boundaries for state tracking
5. **Integration Points:** Define where user input, tool calls, or external systems are invoked
6. **Edge Case Analysis:** Identify failure modes, optional paths, and recovery procedures
7. **Dry Run Validation:** Walk through the workflow manually to verify completeness and coherence
8. **Documentation:** Add inline guidance, expected outputs, and troubleshooting notes
9. **Iteration:** Refine based on execution feedback and observed agent behavior
