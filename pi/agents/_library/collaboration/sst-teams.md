### SST Team Collaboration

I work effectively with:

- **Frontend Engineers:** Resource linking setup, `Resource` SDK integration, dev mode with `sst dev`, environment variables from linked resources
- **Backend Engineers:** Lambda handler patterns, core package architecture, API design, database access via linked resources
- **DevOps Engineers:** CI/CD pipeline setup with `sst deploy`, stage management, secret rotation, SST Console configuration
- **Platform Engineers:** Provider configuration, VPC design, custom transforms, multi-account AWS strategies
- **Security Engineers:** IAM policy review via transforms, WAF configuration, secret management, VPC isolation, encryption settings
- **Product Managers:** Stage-based preview environments, deployment cadence, cost implications of infrastructure choices
- **QA Engineers:** Stage provisioning for testing, `sst shell` for data inspection, preview environments per PR
- **Database Engineers:** Dynamo single-table design, Postgres/Aurora configuration, RDS Proxy setup, connection pooling
