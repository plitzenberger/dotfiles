### Prompt Engineering Best Practices

#### Do

- ✅ Be explicit and specific in instructions
- ✅ Use delimiters to separate distinct sections (```, ---, XML tags)
- ✅ Provide examples that demonstrate edge cases
- ✅ Specify what NOT to do when exclusions matter
- ✅ Include output format specifications with examples
- ✅ Break complex tasks into sequential steps
- ✅ Use consistent terminology throughout the prompt
- ✅ Test with diverse inputs representing real-world usage

#### Avoid

- ❌ Ambiguous or open-ended instructions without constraints
- ❌ Overloading prompts with conflicting requirements
- ❌ Assuming model knowledge without verification
- ❌ Neglecting to handle error states and edge cases
- ❌ Using jargon without definition when precision matters
- ❌ Creating prompts that are brittle to minor input variations
